# Abstract
I used Lua which is a interpreter based scripting language. This language focuses on "Mechanisms over Policies" which means Lua provided the ability to implement required functionality for the project but not necessarily as a standard library feature. Also Lua has some unique quirks that separate it from other modern languages.
# Approach
The first thing I did was to create a main function to house all program execution.

Then I created the 3 subprograms that were listed in the homework instructions: `buildSocialSentimentTable(...)`, `getSocialSentimentScore(...)`, and `getStarRating(...)`. I broke up `getSocialSentimentScore(...)` into two other functions because I felt like it would have done too much on its own to be reasonably called a `get` function.

My algorithm for parsing the `socialsent.csv` file works by finding patterns in the file and then parsing the line one-by-one. Lua does not have POSIX pattern matching because the implementation for such functionality would be larger than the entire Lua standard library, but their pattern matching was more than enough to parse the csv and txt files. So `buildSocialSentimentTable(...)` was mainly a for loop iterating over each line in the csv file. Also this function removes the first line in the csv file, so if testing a csv file that does not have that `word,score,,standford link` first line then the first `<word, score>` pair will be removed.

I created two new functions for `getSocialSentimentScore(...)` called `generateWordScores(...)` and `calculateSocialSentimentScore(...)`. The first function of the two basically parsed each word one-by-one from the input txt file - given by the user - by reading in the whole file and looping over the number of characters present. Set a word variable if a word was found, then find track the number of occurences for that word. Doing this allowed me to handle repeated words in a text to get a more "accurate" result for the social sentiment score of the text. This function mainly created a table containg words and the number of occurences for the word in the input text file. Then, `calculateSocialSentimentScore(...)` calculated the actual sentiment score of the entire input file by looping through the previously created table of words and occurrences. And printing, writing (to a file), and calculating the accumalted sentiment score. This function returned the accumulated social sentiment score which the parent function `getSocialSentimentScore(...)` would assign to the global variable.

The last function was `getStarRating(...)` which used an if-elseif-else statement set to calculate the star rating. Lua has no switch/case statement. I simply used the arbitrary scoring system provided by the homework instructions.
Then the rest of the main function printed and wrote the details of the social sentiment analysis to the console and the output file.
# New Things Learned
There were many things that were different than languages I have learned. All the languages I have learned so far (even a little bit) have all had some definition for creating a class and member methods/variables. Lua does not have this explicit capability. Instead they provide the mechanism for creating classes through tables and metatables. I did not use this functionality for this assignment but I did research it previously. Also Lua starting indexing at `1` instead of `0`. Other interesting thing is that the "does not equal" statement is NOT `!=`, it is actually `~=`. Also, Lua does not have compound statements so all incrementation of numbers must be done explicitly like `count = count + 1`.

I like how quick it was to write Lua. Conditional operators statement control blocks were all defined in words which was good for consistency. I also thought that it was just fun to program in Lua. It felt nice to use and easy to read and understand after coming back to the code. I also like how variable can be explicitly defined as `local` to keep the scope within the current code block. Also I did not use closures for this assignment but I am sure they would have been interesting to use.

What I didn't like about lua was the dynamic typing. I am used to static typing so not being able to define a variable explicitly as an integer is kind of annoying. Also, while I like their "Mechanisms over Policies" ideaology, it literally means that many things need to be created from scratch as a lot of functionality (especially for string handling and manipulation) simply does not exist in the standard library. However I am more neutral towards this idea than negative. I simply have not created a big enough project to get "annoyed" at this feature.
